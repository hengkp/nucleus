Key findings

Methodology & notes

How measured. 25 sequential HTTPS requests per endpoint from node1 via the live nginx edge (TLS + proxy + first byte), using curl timing. The "main" apps and "edge" endpoints traverse the full nginx → (Authelia) → upstream path a real browser uses; "AppHub API" and "Hosted app direct" hit the backend/compute node without the edge to isolate the proxy overhead.

What the numbers are. A first/unauthenticated request typically returns a 302 (to login) or the SPA HTML — this reflects edge + app responsiveness, not the heavier in‑app work (a JupyterHub server spawn, a Galaxy job). Those are application‑internal and out of scope for this network‑latency report.

Caveats. Galaxy returned 502 during the run (its upstream wasn't responding — worth a separate health check). A throttled 84 GB conda sync to shared NFS was running in the background, so absolute numbers are a mild upper bound. Single‑host vantage (node1); real client latency adds WAN/Wi‑Fi.
