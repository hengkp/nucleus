How to read this

What's measured. Time to first byte and total time for the first HTTPS request a browser makes, over the live edge (TLS + nginx + any auth redirect). The hosted "edge" path goes through the per‑instance *.app.sisp.com vhost + Authelia SSO; the hosted "raw app" path hits the container directly on its compute node to isolate the app itself.

What it isn't. This is network/proxy responsiveness, not in‑app work — opening a notebook, spawning a kernel, or an R session is separate and dominated by the app, not the path. A 302 here is the expected first response for all four endpoints.

Takeaway. The new hosting path is faster to first byte than the legacy apps, and the gateway+SSO overhead is small (single‑digit ms). The raw hosted app answers in ~1 ms, so almost all of the hosted edge time is TLS + SSO, not the app.
