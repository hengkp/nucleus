# SISP AppHub — Architecture Decision Records

## ADR-001 — Authentication & Authorization Model (dual credential plane + fail-closed gateway)
**Context.** ~55 non-technical users need one identity/password across a web portal and an SMB file gateway. node1 hosts both OpenLDAP and the public nginx. The file gateway authenticates over SMB/NTLM (port 445), which cannot consume an HTTP cookie. posixAccount stores only crypt `userPassword`, no NT hash.
**Decision.** "SSO" = single identity + single password, NOT a single cross-protocol token. Two credential planes: (a) HTTP — an in-repo `sisp-sso` daemon on a unix socket, gating all web traffic via nginx auth_request, with in-memory ACL + Redis mirror + 5s micro-cache, fail-closed, ≤5s revocation SLA, IP-keyed backoff; (b) SMB — add `sambaSamAccount`/`sambaNTPassword` (Path A) and unify password changes to write both attributes. nginx overwrites X-Remote-User/X-Remote-Groups on every location; auth service ignores inbound identity headers. App vhosts receive header-only identity and NO SSO cookie.
**Alternatives.** Single cross-protocol token (rejected — was the load-bearing error; SMB can't read an HTTP cookie). Kerberos/AD-DC Path B (deferred — changes the entire password story; revisit to eliminate hash-at-rest). Per-request DB/LDAP check (rejected — recreates the concurrency hang). ppolicy hard lockout (rejected — usernames are publicly enumerable → targeted DoS + cluster-wide PAM block). Fail-open (rejected — auth daemon that fails open is a vulnerability).
**Consequences.** NT hash is a second password representation that can drift (mitigated by one transactional set-path). Redis is a new dependency. sisp-sso is a SPOF mitigated by resilience not weakened security. Apps that can't consume X-Remote-User (Zulip, Leantime) keep a second login (edge-gated only).

## ADR-002 — SLURM Job Model (apps as isolated per-user jobs; node1 never scheduled)
**Context.** JupyterHub/RStudio/Galaxy on node1 hang under concurrent load (Problem B). Apps historically bound 0.0.0.0 with auth disabled, reachable by co-scheduled peers. Memory is the binding constraint on 375 GB schedulable.
**Decision.** Each researcher app is a SLURM job on node2-4 running as the real LDAP uid. Payloads bind 127.0.0.1 only, reached via a per-job SSH local-forward from node1, with per-job secret tokens and a host firewall denying 31000-31999 from the LAN; per-job netns where available. Drop the burst partition; `--exclude=node1` everywhere. `persistent` (MaxTime=UNLIMITED) for always-on apps, finite `inter`, network-restricted partition for untrusted, standing small-lane reservation. Event-driven first-login association provisioning. Honest defaults (4 CPU/32 GB single-cell, whole-node pin >64 GB). Control plane protected by systemd cgroups, not SLURM CoreSpec. Galaxy modeled as a persistent DRMAA service, not an idle-culled job.
**Alternatives.** Keep everything on node1 (rejected — the problem). nginx-as-auth-gate for directly-reachable listeners (rejected — no gate when the listener is reachable on loopback). Blanket AdminLevel=Operator on the web account (rejected — web compromise = cluster-wide scancel). Pre-assigned ports (rejected — bind collision; runner picks/verifies port on its node). burst preemption (rejected — inoperative + scheduled the control-plane node).
**Consequences.** Per-job tunnels add control-plane processes + a failure mode. ~6-10 heavy sessions cap → intrinsic queuing. node1 gateway can chokepoint durable I/O. Per-template token discipline is mandatory (CI-gated).

## ADR-003 — Dynamic Routing Strategy (squeue-derived upstreams + debounced map swap + per-app authz)
**Context.** Apps land on dynamically-chosen nodes/ports; the scaffold authenticated but never authorized which app, trusted a job-written status.json host (injection vector), silently truncated names (cross-user misroute), and reloaded nginx inline on every change (serialized launch bursts).
**Decision.** Derive targetHost from `squeue %N` through a hard-coded nodeName→IP table with an IP allowlist (192.168.0.25-28), never from job-written fields. Per-app authorization: nginx forwards X-Apphub-Host to the auth daemon which returns 403 unless owner/share/admin. Reject over-length names (HTTP 400) with an explicit owner field on the route (no slice/truncate). Route publish is fire-and-forget into a debounced syncer with boot-time reconciliation; the swap wrapper uses explicit `if`, flock, fsync, `nginx -t`, unconditional exit 75. Wildcard cert + WebSocket Upgrade headers on every app route. App-domain is a separate label tier (`*.app.<domain>` or a separate registrable domain).
**Alternatives.** Trust status.json host (rejected — injection into root-owned routes.map). Inline sudo+reload per change (rejected — serializes bursts). One-level `*.sisp.com` wildcard for two-label hosts (rejected — doesn't validate `foo.app.sisp.com`). Authenticate-only, no per-app authz (rejected — any user opens any tokenless app).
**Consequences.** Map/store can briefly diverge on crash (boot reconciliation closes it). Port-recycle misroute mitigated by quarantine + netns + instance-bound token. Reload debounce trades a few seconds of activation latency.

## ADR-004 — CIFS Gateway & File Ownership (re-export with mode-based isolation, off node1)
**Context.** Clients mounting the NAS directly leave stale SMB sessions and scramble ownership (Problem A). The fix requires node-side chown/idmap that maps to LDAP uid/gid.
**Decision.** Run the Samba gateway on a dedicated node (recommended node2), NOT node1. NFSv3 AUTH_SYS primary backhaul (raw integer ownership); NFSv4 only with `nfs4_disable_idmapping=Y` gated on a pilot stat-check. Per-user shares isolated by mode (2700/0600, `valid users=%U`); group-write only on explicit `_shared/`. chown/provisioning/remediation run only on the gateway against a node-scoped, firewalled `no_root_squash` export; all other nodes keep root_squash. Samba passdb binds a local read-only syncrepl consumer. Pre-create homes out-of-band as correct uid/gid (no pam_mkhomedir). oplocks/leases off; job output in a separate read-after-completion workspace; Jupyter/RStudio volatile state on node-local /scratch (HOME not on NFS).
**Alternatives.** Keep direct client→NAS mounts (rejected — the problem). Gateway on node1 (rejected — combines files+identity SPOF, recreates Problem B). force group=sisp + 2770/0660 (rejected — made all 55 lockers mutually readable). pam_mkhomedir on the re-export (rejected — root squashed over the double hop → wrong-owner homes). HOME on NFS (rejected — SQLite-over-NFS lock contention).
**Consequences.** AUTH_SYS trusts host-root of the 4 nodes (documented). NAS may not support per-host no_root_squash (Phase-0 gate). Header-trust front door forces an explicit SMB-enrollment campaign. Single gateway is a file-access SPOF (warm standby is a future step).

## ADR-005 — Backend Stack (clean rebuild; Node control plane, fail-closed wrapper, single-writer reconciler)
**Context.** Decision (1) is a full clean rebuild informed by the Codex scaffold. The scaffold's sbatch wrapper had a fallback chain allowing forged-root submission; GET /api/apps reconciled per-request; ports were allocated check-then-act; hot-path state lived on CIFS.
**Decision.** Node server (server.js/lib) on node1 behind the auth gateway, bound to a unix socket with a shared X-Auth-Proxy secret, APPHUB_DEV_AUTH=0 enforced. Fail-closed `apphub-sbatch-as-user.sh` (refuse uid<10000 or gid≠100000, assert getent, pass numeric uid/gid, no head/owned-path fallbacks, reserved usernames blocked in sudoers). Cached non-reconciling GET /api/apps + single-flight reconcile + one batched squeue/tick + debounced nginx reload. Atomic port allocator via partial-unique index ON CONFLICT retry. Idempotent wrapper + job adoption on recovery. Hot-path status on local /var/lib (NOT CIFS); readiness via TCP probe not file mtime. Reverse-reconcile pass. Persisted stuck-clock in Postgres. Postgres on node1 with restore-verified nightly pg_dump (no WAL-to-CIFS).
**Alternatives.** Keep the scaffold as-is (rejected — false "cheap read" premise, security holes). Per-instance squeue/scontrol (rejected — O(N) forks against single-threaded slurmctld). Multi-writer reconciler (rejected — race on routes.map; writer intentionally singular). PITR WAL archiving to CIFS (rejected — blocked WAL recycle → ENOSPC halts the DB).
**Consequences.** Single-writer reconciler is a SPOF (apps keep serving from last map; documented restart runbook). Short auth-cache TTL can briefly honor revoked users. Reload debounce adds small activation latency.

## ADR-006 — Frontend Stack (clean Vite/React rebuild on the approved design language; poll-first, contract-bound)
**Context.** Decision (1)+(2): clean rebuild on "Linear bones, Stripe warmth", SISP teal #11695f, Inter + JetBrains Mono self-hosted, Remix Icon, light-default/dark-available, calm-by-default. The backend has no push channel, reconciles on a ~30s timer, /api/session always returns 200, createAppRecord silently clamps resources and rejects public visibility, and CSP is strict (script-src 'self').
**Decision.** SPA declares a read-path contract the rebuilt backend must satisfy (cached read + single-flight reconcile + debounced reload). Adaptive state-aware polling behind one `live.ts` seam (SSE as a phase-2 swap); aggressive cadence gated behind `VITE_APPS_READ_IS_CHEAP` (default OFF → 30s/60s). Auth bootstrap on session.authenticated===false → full-page redirect to a configured gateway login URL. Externalized hashed theme-boot asset for no-FOUC under nonce-free strict CSP. Tree-shaken inline SVG icons + subset fonts with separate JS/font/CSS budgets. Clamp client-side + seed cache from the returned app; private/team only. Node gauges conditional on a cached GET /api/cluster/nodes with graceful degradation. zod safeParse with lenient optionals; StaleBadge when age >2× interval.
**Alternatives.** WebSocket/SSE for v1 (deferred — no backend push; one-module swap later). Trust a fetch-interceptable 401 from the gateway (rejected — gateways return Basic/HTML 302 that break fetch JSON). Inline theme bootstrap script (rejected — blocked by CSP → FOUC). Full ri-* webfont + unsubset fonts (rejected — understates first paint on lab networks). Typed resource inputs / public option (rejected — would lie about granted resources / hard-fail 400).
**Consequences.** If the backend cached-read isn't shipped, the SPA runs conservative cadence (still a hazard on any reconcile-on-read path — blocking ticket). Exact gateway auth module + login/logout URLs must be pinned before auth.ts is finalized.
