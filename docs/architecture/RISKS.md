# SISP AppHub — Risk Register

Severity ranked High → Low. De-duplicated across subsystems (e.g. the node1-SPOF, *.app domain, untrusted-app loopback, and auth-SPOF themes were stated by multiple subsystems and are merged here).

| # | Sev | Risk (consolidated) | Mitigation | Owner |
|---|-----|---------------------|------------|-------|
| 1 | **High** | NAS may not preserve numeric ownership through the gateway (NFSv4 idmap → `nobody:nobody`) or may forbid per-host `no_root_squash`, breaking the entire uid/gid guarantee and provisioning. | Phase-0 Spike S1 hard gate: `stat` 10012→10012 before any reliance; prefer NFSv3 AUTH_SYS; documented NAS-side provisioning-API fallback. | Storage |
| 2 | **High** | node1 is a single point of failure for LDAP + nginx + auth + Postgres + CIFS gateway candidate + all WebSocket proxying; its exhaustion takes down everything. | Move CIFS gateway to node2; systemd cgroup caps on node1 services; restore-verified nightly dumps + off-node config; documented rebuild runbook; HA noted as out-of-scope cluster decision. | Ops + Architecture |
| 3 | **High** | Untrusted/co-located SLURM apps share the host netns; if per-job isolation is misconfigured a co-tenant reaches a peer's listener on loopback as that uid. | Native per-app token is a HARD requirement (never token-disabled) so a netns gap degrades to authenticated-listener; dedicated network-restricted partition not co-tenanted with interactive jobs; co-tenant-unreachable smoke test fails the rollout. | Scheduler + Security |
| 4 | **High** | X-Remote-User header-trust is the identity root; a misconfigured/passthrough header from a public hop spoofs identity despite the fail-closed wrapper. | nginx overwrites/clears X-Remote-* on every location; auth service ignores inbound identity; wrapper refuses uid<10000 + asserts LDAP numeric uid/gid; header-injection test in deploy gate. | Auth + Backend |
| 5 | **High** | SSO cookie scoped to `.sisp.com` is shipped to owner-controlled/third-party app backends → cross-app credential/session theft. | Serve apps on a separate registrable domain and never send SSO cookie there; mint short-lived instance-bound app token; until domain procured, `__Host-` cookie + strict CSP as documented partial fix. | Security + Networking |
| 6 | **High** | A template regressing to `--auth-none`/empty token/0.0.0.0 reopens an unauthenticated kernel/shell running as the owner. | CI gate on templates.json forbidding 0.0.0.0/empty-token/--auth-none; fix in all three locations (rstudio.def, jupyterlab, rstudio template); authz + loopback-bind smoke test. | Scheduler + Security |
| 7 | **High** | sisp-sso fails closed and fronts 100% of web traffic → total web outage if it and the micro-cache both lapse; also a concurrency hot-spot. | In-memory ACL + caches keep hot path off LDAP/Postgres; Restart=always + watchdog + /healthz + p99 alerting; >100-session load test as go-live gate; fail-closed accepted as deliberate security posture. | Auth |
| 8 | **High** | Frontend goes live before backend ships cached non-reconciling read + single-flight reconcile + debounced reload → 5s polling by 30 users recreates the concurrency hang (Problem B). | Aggressive cadence hard-gated behind default-off flag (falls back to 30s/60s); cached-read/mutex/reload-debounce filed as blocking launch tickets; StaleBadge surfaces lag. | Frontend + Backend |
| 9 | Medium | AUTH_SYS gives no client crypto auth; any compromised node-root in .25-.28 can forge a uid and read/write any locker. | Firewall NAS export to the 4 node IPs; root_squash on compute nodes; Apptainer dropped caps/no_new_privs; sec=krb5 if NAS supports; residual trust = host-root of 4 nodes, documented. | Storage + Security |
| 10 | Medium | NT-hash (`sambaNTPassword`) is a second password representation that can drift from `userPassword` if any change bypasses the unified path. | Single transactional set-password action writing both attributes; audit for users missing the hash; Path B (Kerberos) considered to eliminate hash-at-rest. | Auth |
| 11 | Medium | Cleartext LDAP bind from node2-4 until internal CA + StartTLS deployed; LAN is the only trust boundary during bootstrap. | Treat `reqcert=allow` as dated bootstrap; prioritize CA; firewall 389 to cluster subnet in the interim. | Auth + PKI |
| 12 | Medium | Memory is the binding constraint: ~6-10 concurrent heavy single-cell sessions exhaust schedulable RAM → users queue. | Honest queue UI with estimated start; small-lane reservation; whole-node pinning to prevent thrash; written "heavy work queues" policy; only more RAM/nodes raises ceiling. | Scheduler |
| 13 | Medium | node1 NFSv4 re-export / CIFS gateway is a throughput chokepoint for durable workspace I/O under concurrent heavy sessions. | Offload temp/heavy I/O to node-local /scratch + prewarmed images; benchmark gateway throughput (assume ≥10GbE); evaluate per-node mounts if it saturates. | Storage + Scheduler |
| 14 | Medium | slurmdbd + MariaDB on node1 add load + SPOF to the protected node; accounting enforcement depends on them. | Place under control-plane cgroup budget, size from measured load, back up DB; relocating to node2 is an open question pending measured load. | Scheduler |
| 15 | Medium | Auth-decision caching can honor a just-revoked user up to the short TTL (~5s edge, ~60s idle after a crash). | No cache on admin routes (first-request revoke); document the 5s/60s bounds; admin force-logout rotates signing key. | Auth + Backend |
| 16 | Medium | Apptainer mode (userns vs setuid) on node2-4 is unconfirmed and changes the entire container/network threat model. | Verify mode as a documented precondition before relying on userns isolation; revisit isolation flags if setuid found. | Security + Container |
| 17 | Medium | DNS-01 wildcard renewal depends on an external provider API; outage/misconfig in the window breaks all app URLs at once. | Independent A/B certs; alerts at 30/21/14 days; CNAME-delegated minimal-privilege zone; documented manual issuance runbook. | Networking + PKI |

| 19 | Medium | Single CIFS gateway node is a SPOF for all desktop file access. | systemd slice isolation, restart limits, RPC-timeout monitoring, SLURM core reservation; warm-standby gateway on node3 documented as future step. | Storage |

| 21 | Medium | Per-job SSH tunnels add control-plane process load and a failure mode (tunnel dies, job lives); netns/veth setup may exceed the confirmed-reload boundary. | Count tunnels vs running jobs; reconciler re-establishes missing tunnels; tunnels within control-plane cgroup; confirm SSH-key trust acceptable (open question). | Scheduler + Networking |
